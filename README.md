# 🔐 tiny-xor

**Blazing fast, zero-dependency file encryption tool written in pure x86_64 Assembly.**

## ✨ Особенности

- 🚀 **Микроскопический размер:** Бинарный файл весит ~10 КБ (для сравнения, аналог на C с libc весит 50+ КБ).
- ⚡ **Ноль зависимостей:** Нет libc, нет рантайма. Программа общается напрямую с ядром Linux через системные вызовы (syscalls).
- 📦 **Экономия памяти:** Обрабатывает файлы чанками по 4 КБ. Можно зашифровать файл на 100 ГБ, не съев всю оперативную память.
- 🔐 **Симметричное шифрование:** Одна и та же команда и шифрует, и расшифровывает файл.
- 🎯 **Чистый ассемблер:** Никаких абстракций языков высокого уровня, максимальный контроль над железом.

## 🛠️ Сборка

Для сборки вам понадобится NASM и стандартный линковщик Linux:

```bash
nasm -f elf64 xor.asm -o xor.o
ld -o tiny-xor xor.o
```

## 📖 Использование

```bash
./tiny-xor <входной_файл> <выходной_файл> <ключ>
```

### Примеры

**Зашифровать файл:**
```bash
./tiny-xor secret.txt encrypted.bin my_super_secret_key
```

**Расшифровать файл (тем же ключом):**
```bash
./tiny-xor encrypted.bin decrypted.txt my_super_secret_key
```

## ⚠️ Примечание о безопасности

Этот инструмент использует XOR-шифрование, которое **не является криптографически стойким** для защиты конфиденциальных данных от целевых атак. Он отлично подходит для:
- Обфускации (скрытия) некритичных файлов
- Обучения (изучение ассемблера и системного программирования)
- Быстрого "перемешивания" данных

Для настоящего шифрования используйте проверенные инструменты: `gpg`, `openssl` или `age`.

## 🧪 Тестирование

```bash
# Создаём тестовый файл
echo "Secret message" > test.txt

# Шифруем
./tiny-xor test.txt encrypted.bin test_key

# Расшифровываем
./tiny-xor encrypted.bin decrypted.txt test_key

# Проверяем, что данные совпадают
diff test.txt decrypted.txt && echo "✅ Успех! Файлы идентичны."
```
